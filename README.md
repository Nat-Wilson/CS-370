# CS-370
Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?

For this project, I was given a basic maze environment (TreasureMaze) and some helper classes for managing game state. From there, I created the core learning components myself. This included building a neural network model for the Q-learning agent, implementing experience replay, coding the training loop with an epsilon-greedy policy, and tracking the agent’s performance over time. I also handled preprocessing the environment states so they could be used as inputs for the model.

What do computer scientists do and why does it matter?

Computer scientists design algorithms and build systems that solve real-world problems efficiently. They analyze data, create software, and develop models that can automate tasks or provide insights. This work matters because it drives innovation across industries, improves decision-making, and allows people and organizations to solve problems faster and more accurately than they could manually.

How do I approach a problem as a computer scientist?

As a computer scientist, I approach a problem by breaking it into smaller, manageable pieces. I analyze the requirements, design a solution, implement it, and test it repeatedly to improve performance. In this project, that meant identifying the environment, defining the agent’s inputs and outputs, implementing learning rules, and iterating on the model and training process until the agent could navigate the maze successfully.
